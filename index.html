Introduction

If there is one thing that people seem to struggle with a lot in Javascript, it's object inheritance. Since Javascript isn't an object oriented language but prototype based, it's not very easy to write something in an object oriented fashion. This library is lightweight (a little under 4kb minified) with a sole purpose of allowing you, the developer, to easily develop applications in an object oriented matter.

Features

  • Object- and function based class bodies
  • Extending not only JOII- but also native-objects
  • Implementing interfaces
  • Implementing traits
  • Adding mixins on-the-fly
  • Using constructors and destructors
  • Overrides & parent inheritance
  • Using aliases to beat the scoping troubles
  • Supports even Internet Explorer 6

Usage

JOII comes with a Class function which you use to declare classes. This function accepts 2 arguments: an options object and a body function or object. The options object is optional and should only be used if you really need it.

var MyClass = Class(function() {
    // Class without options.
});

var MyOtherClass = Class({ extends: MyClass }, function() {
    // Class with options.
});

Object vs Function

Javascript has objects and functions. The difference between an object and a function is mainly that an object, declared as an actual object, is stateless where a function is not. When using JOII for object oriented programming, it's very important that you're aware of these differences.

var obj = {
    hello: "I am an object",
    world: "Easy no?"
}

var func = function() {
    this.hello = "I am a function";
    this.world = "Easy no?"
}

The reason for this being important is that some elements may only be implemented as objects where others can only be implemented as functions. For example, a trait is stateless because it must only contain methods which are copied to the designated class. Therefore, a trait can only be declared as an object instead of a function.

Usage

Extending classes

Extending classes is very easy. Simply pass the object, function or class you wish to extend on in the options of your own class declaration, like so:

var MyClass = Class({ extends: AnotherClass }, function() {
    // This class now inherits the properties and methods of 'AnotherClass'
});

You can also extend classes upon classes upon classes...

var a = Class(function() {
    this.a = 1;
});
var b = Class({ extends: a }, function() {
    this.b = 2;
});
var c = Class({ extends: b }, function() {
    this.c = 3;
});
var foo = Class({ extends: c }, function() {
    this.__construct = function()
    {
        // print: a = 1, b = 2, c = 3
        for (var i in this) {
            console.log(i + ' = ' + this[i]);
        }
    }
});

Implementing Interfaces

Object interfaces allow you to create code which specifies which methods a class must implement, without having to define how these methods are handled. Interfaces are defined using the Interface function, the same way as a class is defined.

An interface may only contain public properties which define their type. An interface may not contain any functional code. This is the nature of an interface.


Note:
Interfaces may extend on other interfaces, the same way as classes extend on other classes, but a class may not extend on an interface and vice-versa.

Implementation example

An interface is defined as such:

var iPerson = Interface({
    getName: 'function',
    getEmail: 'function'
});

Top implement an interface, the implements option is used.

var Person = Class({ implements: [iPerson]}, function() {
    this.getName = function() {
        return "John Smith";
    }
});

var p = new Person();
// Will throw an exception: "Missing function implementation: getEmail".

To verify if a class implements a certain interface, the implements method is used. This method is available by default to any class defined as a JOII-class.

if (p.implements(iPerson)) {
    console.log(p.getName());
}

Traits & Mix-ins

JOII works purposely as a single inheritance system. To solve the horizontal code reuse problem, we give you Traits.

A Trait is intended to reduce some limitations of single inheritance by enabling a developer to reuse sets of methods freely in several independent classes living in different class hierarchies. The semantics of the combination of Traits and classes is defined in a way which reduces complexity, and avoids the typical problems associated with multiple inheritance and Mixins.

The difference between a trait and a mix-in is that a trait can solely be declared as an object, where a mixin may be a function which hasn't even been instantiated yet.

A simple trait example

var tMath = {
    add: function(a, b) { return a + b; },
    sub: function(a, b) { return a - b; }
}

To implement this trait into your class, use the uses option. The same as implements, uses also only accepts an array of values, like so:

var MyClass = Class({ uses: [tMath] }, function() {

    this.__construct = function() {
        return this.add(2, 4);  // returns: 6
    }

});

Mix-in

A mix-in is an on-the-fly injection mechanism to quickly apply logical code to an existing (pre-initialized) class. A usecase for this is debugging an existing object with a "debugger" mix-in, which contains a set of debugging tools.

var MyMixIn = Class(function() {
    // Prints the property 'foo' from whichever class we're mixed into. 
    this.mix = function() {
        console.log(this.foo);
    }
});

var MyClass = Class(function() {
    this.foo = "bar";
});

var m = new MyClass();
m.mixin(MyMixIn);
m.mix(); // prints: bar
Note:
There is no restriction on the object-type of a mix-in, as long as it can be iterated over to "copy" elements from the mix-in to the designated class.

Aliases

Since you might not always have access to the scope where your classes are declared in, it's possible to give your classes an alias which can be used from anywhere. Defining an alias is done using the name option. You can then use this name in extends or implements or uses by simply passing a string value instead of the direct object.

var BaseClass = Class({ name: 'example.class' }, function() {
    // A class with an alias.
});

// Normal inheritance, extending on the direct object.
var MyClass = Class({ extends: BaseClass }, function() { });

// Aliased inheritance, extending on the alias.
var MyClass = Class({ extends: 'example.class' }, function() { });

Basic usage example

In this example, we'll be creating a set of classes that allow us to create some elements on our website. We'll do this by creating a Renderer class which renders the HTML, a logger trait and an interface.

var Example = {};

// Renderer interface
Example.RendererInterface = Interface({
    /**
     * Create an element and append it to the body of the document.
     *
     * @param string type       div, span, section, etc.
     * @param string class      css-class
     * @param string data       element contents
     *
     * @return DOMElement
     */
     addElement: 'function'
});

// Renderer class
Example.Renderer = Class({ implements: [Example.RendererInterface] }, function() {

    this.render_target = null;
    /**
     * @see Example.RendererInterface.createElement
     */
    this.addElement = function(type, class_name, data)
    {
        var el = document.createElement(type);
        el.setAttribute('class', class_name);
        el.innerHTML = data;
        this.render_target.appendChild(el);
    }
});

// Logger
Example.LoggerTrait = {
    log   : function(msg) { console.log(msg); },
    warn  : function(msg) { console.warn(msg); },
    error : function(msg) { console.error(msg); },
}

// Presentation Class
Example.Presentation = Class({ extends: Example.Renderer, uses: [Example.LoggerTrait] }, function() {

    this.__construct = function(target)
    {
        this.render_target = target;
    }

    this.title = function(title) {
        this.log('Created h1 element: ' + title);
        this.addElement('h1', 'title', title);
    }

    this.hr = function() {
        this.log('Created hr element');
        this.addElement('hr', null, null);
    }

    this.paragraph = function(data) {
        this.log('Created paragraph.');
        this.addElement('p', 'paragraph', data);
    }

});

Now that we have a simple renderer, let's put it to the test. We'll instantiate the Example.Presentation class and apply a mix-in to it that will put some things on the screen.

var MakeAwesumStuff = Class(function() {

    this.doThings = function()
    {
        this.title('Hello World!')
        this.hr();
        this.paragraph('Lorum ipsum dollarrr spectacularr ahmet.');
    }

});

// Instantiate Presentation on document.body as render-target.
var presentation = new Example.Presentation(document.body);

// Inject the mix-in
presentation.mixin(MakeAwesumStuff);

// Execute the mix-in function
presentation.doThings();
